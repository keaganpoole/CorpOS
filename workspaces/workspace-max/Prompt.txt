-----------------------------------
VALIDATION / EDITING UX (VERY IMPORTANT)
-----------------------------------

One of the most important parts of this Leads page is that editing must be extremely user friendly and should actively prevent invalid input whenever possible.

This UI should respect the actual Supabase column types and constraints so the user is not allowed to accidentally enter bad data that would fail on save.

In other words:
The frontend should intelligently protect the user from backend write errors.

Examples:
- If a field is numeric, do not allow letters or invalid characters
- If a field is a timestamp/date, use an appropriate date/time input or picker
- If a field is a single-select style field (like status/source), use controlled dropdowns instead of free text
- If a field is an array/multi-value field (like opportunity), use a proper multi-select or token/tag style UI
- If a field should be a URL, guide the user appropriately
- If a field is optional, make that clear
- If a field is required, make that clear
- If input is invalid or restricted, provide subtle, helpful inline guidance

Very important:
Do NOT make the user wonder why something “isn’t working.”

If something is not allowed:
- make it obvious
- make it graceful
- make it subtle but clear

Use:
- inline validation
- helpful helper text
- subtle warnings
- intelligent field controls
- sensible formatting behavior
- input restrictions where appropriate

The goal is:
prevent errors before they happen, not just show Supabase errors after submission.

This page should feel highly polished, forgiving, and confidence-inspiring when editing leads.
